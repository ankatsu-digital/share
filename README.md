# 自分用セキュアファイル共有システム(Google Apps Script + GitHub Pages版)

**構成**: 画面(見た目)は GitHub Pages、データ処理(アップロード・パスワード発行・メール送信・ダウンロード管理)は Google Apps Script が担当します。費用は完全無料です。

```
Code.gs          → script.google.com に貼り付けて「ウェブアプリ」として公開(APIとして動く)
site/            → GitHub Pagesのリポジトリ直下にそのままアップロードする中身(フォルダ分けなし)
  admin.html     → 管理画面。タブ式UI・CSSはこのファイルに埋め込み済み(style.cssは使いません)
  admin.js
  share.html     → 受信者用画面。CSSはstyle.cssを参照
  share.js
  config.js      → ここにApps ScriptのURLを書く
  style.css      → share.html用
```

## セットアップ手順

### 1. Apps Script側(API)を作る

1. https://script.google.com/create で新規プロジェクト作成
2. デフォルトの `コード.gs` を全部消して、このフォルダの `Code.gs` の中身を貼り付ける
3. 「プロジェクトの設定」→「スクリプト プロパティ」に以下を追加:
   - `ADMIN_PASSWORD` : 管理画面ログイン用パスワード
   - `SITE_BASE_URL` : GitHub Pagesの公開URL(例: `https://ユーザー名.github.io/リポジトリ名`)。**手順2でGitHub Pagesを公開したあとに設定してください**
   - `MAIL_REPLY_TO` (任意): 返信先アドレス。**これは管理画面の「送信設定」からいつでも変更できるので、最初は設定しなくてもOK**
   - `MAIL_FROM_ALIAS` (任意): Gmailで認証済みの送信エイリアスがあれば、そこから送信できます(下記「送信元アドレスについて」参照。こちらはスクリプトプロパティでのみ設定可能)
4. 「デプロイ」→「新しいデプロイ」→ 種類「ウェブアプリ」→ 実行:自分 / アクセス:全員 → デプロイ
5. 発行された「ウェブアプリのURL」(`https://script.google.com/macros/s/xxxx/exec`)をコピー

### 2. GitHub Pages側(画面)を作る

1. GitHubで新しいリポジトリ作成(Public)
2. `site/` フォルダの中身(`admin.html` `admin.js` `share.html` `share.js` `config.js` `style.css`)を**フォルダ分けせずリポジトリ直下に**アップロード
3. `config.js` の `GAS_URL` を、手順1-5でコピーしたApps ScriptのURLに書き換える
4. リポジトリの Settings → Pages → Source を「Deploy from a branch」→ Branch: `main` / `root` に設定
5. 公開されたURL(例: `https://ユーザー名.github.io/リポジトリ名`)を、手順1-3の `SITE_BASE_URL` にも設定する(Apps Scriptのスクリプトプロパティを更新)

### 3. 動作確認

1. `https://ユーザー名.github.io/リポジトリ名/admin.html` を開き、パスワードでログイン
2. ファイルと受信者のメールアドレスを入力してアップロード
3. リンクメールが届くか、1分後にパスワードメールが届くかを確認
4. `share.html?id=xxxx` を開いてダウンロードまで一通り確認

## 送信元アドレスについて

Apps Scriptの`MailApp`/`GmailApp`は、なりすまし送信ができない仕様上、**実行しているGoogleアカウント自身のアドレスからしか送信できません**。選択肢は3つです。

1. **返信先だけ変える**(一番簡単・管理画面から設定可能): 管理画面にログインすると「送信設定」というカードがあり、そこに返信先アドレスを入力して保存するだけです。送信元表示はそのままですが、相手が「返信」を押すと指定したアドレスに届きます
2. **送信元アドレス自体を変える**: Gmail設定(歯車→すべての設定を表示→アカウントとインポート→「名前」の項目にある「他のメールアドレスを追加」)で送信エイリアスを追加・確認コード認証したあと、Apps Scriptのスクリプトプロパティに `MAIL_FROM_ALIAS` を追加してそのアドレスを設定すると、そこから送信されます(こちらは管理画面からではなく、スクリプトプロパティで設定します)
3. **アカウントを複数作る**: 送りたいアドレスそれぞれでこのシステム一式(Apps Script + GitHub Pages)を別々に作る。一番確実ですが、その分セットアップの手間も増えます

## 新機能

- **カスタムリンクID**: アップロード時に「リンクID」を自分で指定できます(空欄なら自動生成)。半角英数字・ハイフン・アンダースコアのみ、3〜64文字
- **送信履歴**: 管理画面下部に過去の送信一覧が表示され、状態(有効/無効化済み/期限切れ/キャンセル済み/ダウンロード上限到達)を確認できます
- **手動でのリンク無効化**: 送信履歴の各項目から「無効化」ボタンを押すと、まだ有効な状態(パスワード送信前後・ダウンロード前後を問わず)のリンクをいつでも無効化できます。元に戻せないのでご注意ください

## 仕組み(v2)

- **ファイル保存**: あなたのGoogleドライブ内に自動作成される `SecureFileShare_Storage` フォルダ
- **データ管理**: あなたのGoogleドライブ内に自動作成されるスプレッドシート `SecureFileShare_DB`
- **メール送信**: Gmail経由、無料。アップロードすると**即座に**リンク+ログインIDのメールが自動送信されます(LINE等で手動送信する必要はありません)
- **パスワードの自動送信**: メール送信から**1分後**に、Apps Scriptの時間主導トリガーで自動送信されます(受信者がリンクを開いているかどうかに関係なく届きます)
- **キャンセル**: 管理画面に「送信待ち一覧」があり、パスワード未送信の1分間はキャンセルできます
- **ダウンロード回数・有効期限**: アップロード時に設定可能。認証(ID・パスワード入力)しただけではカウントされず、**実際にダウンロードボタンを押した時**だけ回数が消費されます。リロードしても無効になりません
- **公開URL**: GitHub Pages(見た目)+ Apps Script(処理・API)の組み合わせ

料金は完全無料です(個人利用の範囲であれば)。

## 制限事項(把握しておいてください)

- 消費者向けGoogleアカウントは **1日あたりのメール送信数が100通まで**
- ファイルはBase64でやり取りするため、あまり大きなファイル(目安20〜30MB以上)は不向き
- Apps Scriptの1回の実行時間上限は6分
- 1分後の自動送信は「時間主導トリガー」という仕組みを使っており、実行タイミングが数十秒前後ずれることがあります(ジャストその瞬間ではなく「だいたい1分後」の精度です)
- ブラウザからApps ScriptへのAPI通信(fetch)は、Google側の仕様変更で動作が変わる可能性があります

## セキュリティについて

- ファイルの中身は「ログインID・パスワードの照合に成功した場合のみ」サーバー側(Apps Script)からブラウザへ渡される設計なので、URLだけでファイルを取得することはできません
- 管理者ページのパスワードは平文でスクリプトプロパティに保存されています。よほど厳重にしたい場合を除き、個人利用であれば問題ないレベルです
- スプレッドシート(`SecureFileShare_DB`)やDriveフォルダ(`SecureFileShare_Storage`)は、あなたのGoogleドライブ内に作られ、初期設定では自分以外はアクセスできません
- `config.js` に書くApps ScriptのURLは公開されます(誰でも見える状態になりますが、パスワード認証はサーバー側で行っているので問題ありません)

## コードを更新した場合の注意

`Code.gs` を修正したときは、Apps Scriptで「デプロイ」→「デプロイを管理」→ 該当デプロイの鉛筆アイコン→ バージョンを「新バージョン」にして「デプロイ」し直してください(GAS_URL自体は変わりません)。
